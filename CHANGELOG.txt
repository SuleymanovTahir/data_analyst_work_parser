2026-03-31

ВАЖНО ПОМНИТЬ
- Основная логика поиска вакансий должна жить только в `hh_vacancy_bot.py`.
- Второй entrypoint `hh_vacancy_parser.py` удалён. Для запуска и доработок использовать только `hh_vacancy_bot.py`.
- Для географии нельзя полагаться только на точное сравнение `area_id`: исключение страны или региона должно учитывать дочерние города и области.
- При комбинировании нескольких запросов и вариантов опыта итоговую выдачу нужно сортировать после дедупликации, иначе глобальная сортировка ломается.

ИЗМЕНЕНИЯ
- Доработан Telegram-бот поиска вакансий hh.ru под сложные шаблоны поиска.
- Добавлены настраиваемые поля поиска: название вакансии, компания, описание.
- Добавлен режим опыта `Не имеет значения` плюс мультивыбор всех HH-опций.
- География переделана на две независимые группы: `included_area_*` и `excluded_area_*`.
- Исключение регионов теперь учитывает дочерние города и области, поэтому сценарии вроде `все страны кроме Москвы` и `выбранные страны, кроме отдельных городов` работают корректно.
- Добавлена настройка глубины поиска по страницам.
- Добавлена локальная сортировка по совпадению с запросом и по названию.
- Дефолтный шаблон заменён на `Аналитик — все страны кроме Москвы`.
- Расширены дефолтные синонимы аналитических вакансий и список исключаемых gambling/game-related слов.
- Удалён `hh_vacancy_parser.py`, чтобы в проекте остался только один рабочий файл запуска.
- История отправленных вакансий переведена с общей на по-шаблонную: теперь один шаблон не мешает другому и `preview` не сбивает основную рассылку.
- Добавлены новые команды: `/menu`, `/current`, `/preview`, `/clone`, `/reset_sent`.
- Добавлен быстрый inline-меню экран с основными действиями без ввода команд вручную.
- В wizard добавлены новые фильтры: включающие слова, формат работы, тип занятости, период поиска, лимит результатов, минимальная зарплата, только вакансии с зарплатой, исключение работодателей.
- Предпросмотр поиска теперь не записывает вакансии в историю и не сдвигает таймер автопроверки.
- Форматирование вакансий переведено на безопасный HTML-escape, чтобы спецсимволы в названиях и компаниях не ломали Telegram-сообщения.
- Подтянуты словари HH (`employment`, `work_format`) для актуальных опций прямо из API.
- Telegram-токен вынесен из `hh_vacancy_bot.py` в локальный файл `bot_local_config.py`, а сам файл добавлен в `.gitignore`.
- Добавлен полноценный webhook-режим: бот умеет сам регистрировать webhook в Telegram, принимать POST через встроенный HTTP-сервер и продолжать плановый поиск без polling.
- Добавлена Vercel-совместимая serverless-обвязка через FastAPI и `api/[[...path]].py`: бот теперь можно деплоить на Vercel без отдельного локального процесса.
- Конфиг для Vercel переведён на переменные окружения `HH_TELEGRAM_BOT_TOKEN`, `HH_TELEGRAM_WEBHOOK_URL`, `CRON_SECRET`; локальный `bot_local_config.py` на Vercel больше не используется.
- Данные состояния на Vercel переведены на `Runtime Cache`, а тяжёлое дерево регионов HH сжимается через gzip+base64 перед записью в cache.
- Исправлен баг serverless-режима: дефолтный шаблон теперь создаётся не только при локальном старте, но и при первой загрузке состояния на Vercel, поэтому прод не остаётся пустым (`templates_count = 0`).
- Добавлены публичные Vercel endpoints: `/`, `/status`, `/webhook-info`, `/webhook-register`, `/telegram-webhook`, `/cron`, `/run-now`.
- Для бесплатного Vercel Hobby зафиксирован рабочий сценарий: webhook + ручной `/run`; частый встроенный cron не считается основной логикой автопоиска.
- Интерфейс бота упрощён: в главном меню оставлены только понятные разделы, термин `шаблон` в пользовательских экранах заменён на `поиск`, а логика копирования поиска убрана из меню и команд.
- Выдача вакансий больше не отправляется потоком: результаты собираются в постраничную подборку с перелистыванием `Назад` / `Далее`, по умолчанию по 5 вакансий на страницу.
- В мастер поиска добавлены навигационные кнопки `Назад` и `Главное меню` почти на каждом шаге, а формат работы можно выбирать не только кнопками, но и вводом номеров пунктов.
- Сообщения о пустом результате стали понятнее: бот теперь различает `ничего не найдено`, `всё уже отправлялось раньше` и ошибки запроса к HH API.
- Дефолтный поиск уточнён: теперь это `Аналитик — все страны кроме России`, а не некорректная формулировка про Москву.
- Добавлен бесплатный внешний scheduler через GitHub Actions: workflow `.github/workflows/hh-search-scheduler.yml` вызывает Vercel endpoint `/cron` каждые 5 минут и умеет ручной `workflow_dispatch`.
- Workflow считает ошибкой только реальные сбои (`HTTP >= 400` или `status=error`), а штатные пропуски вроде `interval_not_reached` и `inactive_or_chat_missing` не помечает как падение.
- На Vercel добавлена полноценная веб-панель по корневому адресу сайта: теперь поиск можно создавать, редактировать, сохранять, удалять, делать текущим, ставить на паузу, запускать и просматривать прямо из браузера без Telegram.
- Веб-панель встроена прямо в `hh_vacancy_bot.py` через FastAPI + HTML/JS, без отдельного фронтенд-проекта и без дублирования поисковой логики.
- Для сайта добавлены JSON endpoints `/api/web/*`: состояние, сохранение поиска, выбор текущего, удаление, очистка истории отправок, переключение статуса и запуск поиска/предпросмотра.
- Корень Vercel `/` и страницы `/ui`, `/index`, `/index.html` теперь открывают веб-панель, а служебные JSON endpoints оставлены на `/status`, `/health`, `/api/*`.
