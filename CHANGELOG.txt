ВАЖНО ПОМНИТЬ
- Основная логика поиска вакансий должна жить только в `hh_vacancy_bot.py`.
- Второй entrypoint `hh_vacancy_parser.py` удалён. Для запуска и доработок использовать только `hh_vacancy_bot.py`.
- Для географии нельзя полагаться только на точное сравнение `area_id`: исключение страны или региона должно учитывать дочерние города и области.
- При комбинировании нескольких запросов и вариантов опыта итоговую выдачу нужно сортировать после дедупликации, иначе глобальная сортировка ломается.

- Добавлена настройка глубины поиска по страницам.

- Расширены дефолтные синонимы аналитических вакансий и список исключаемых gambling/game-related слов.

- История отправленных вакансий переведена с общей на по-шаблонную: теперь один шаблон не мешает другому и `preview` не сбивает основную рассылку.

- В wizard добавлены новые фильтры: включающие слова, формат работы, тип занятости, период поиска, лимит результатов, минимальная зарплата, только вакансии с зарплатой, исключение работодателей.
- Предпросмотр поиска теперь не записывает вакансии в историю и не сдвигает таймер автопроверки.
- Форматирование вакансий переведено на безопасный HTML-escape, чтобы спецсимволы в названиях и компаниях не ломали Telegram-сообщения.
- Подтянуты словари HH (`employment`, `work_format`) для актуальных опций прямо из API.
- Telegram-токен вынесен из `hh_vacancy_bot.py` в локальный файл `bot_local_config.py`, а сам файл добавлен в `.gitignore`.
- Добавлен полноценный webhook-режим: бот умеет сам регистрировать webhook в Telegram, принимать POST через встроенный HTTP-сервер и продолжать плановый поиск без polling.
- Добавлена Vercel-совместимая serverless-обвязка через FastAPI и `api/[[...path]].py`: бот теперь можно деплоить на Vercel без отдельного локального процесса.
- Конфиг для Vercel переведён на переменные окружения `HH_TELEGRAM_BOT_TOKEN`, `HH_TELEGRAM_WEBHOOK_URL`, `CRON_SECRET`; локальный `bot_local_config.py` на Vercel больше не используется.
- Данные состояния на Vercel переведены на `Runtime Cache`, а тяжёлое дерево регионов HH сжимается через gzip+base64 перед записью в cache.

- Добавлены публичные Vercel endpoints: `/`, `/status`, `/webhook-info`, `/webhook-register`, `/telegram-webhook`, `/cron`, `/run-now`.
- Для бесплатного Vercel Hobby зафиксирован рабочий сценарий: webhook + ручной `/run`; частый встроенный cron не считается основной логикой автопоиска.
- Интерфейс бота упрощён: в главном меню оставлены только понятные разделы, термин `шаблон` в пользовательских экранах заменён на `поиск`, а логика копирования поиска убрана из меню и команд.
- Выдача вакансий больше не отправляется потоком: результаты собираются в постраничную подборку с перелистыванием `Назад` / `Далее`, по умолчанию по 5 вакансий на страницу.
- В мастер поиска добавлены навигационные кнопки `Назад` и `Главное меню` почти на каждом шаге, а формат работы можно выбирать не только кнопками, но и вводом номеров пунктов.
- Сообщения о пустом результате стали понятнее: бот теперь различает `ничего не найдено`, `всё уже отправлялось раньше` и ошибки запроса к HH API.
- Дефолтный поиск уточнён: теперь это `Аналитик — все страны кроме России`, а не некорректная формулировка про Москву.
- Добавлен бесплатный внешний scheduler через GitHub Actions: workflow `.github/workflows/hh-search-scheduler.yml` вызывает Vercel endpoint `/cron` каждые 5 минут и умеет ручной `workflow_dispatch`.
- Workflow считает ошибкой только реальные сбои (`HTTP >= 400` или `status=error`), а штатные пропуски вроде `interval_not_reached` и `inactive_or_chat_missing` не помечает как падение.
- На Vercel добавлена полноценная веб-панель по корневому адресу сайта: теперь поиск можно создавать, редактировать, сохранять, удалять, делать текущим, ставить на паузу, запускать и просматривать прямо из браузера без Telegram.
- Веб-панель встроена прямо в `hh_vacancy_bot.py` через FastAPI + HTML/JS, без отдельного фронтенд-проекта и без дублирования поисковой логики.
- Для сайта добавлены JSON endpoints `/api/web/*`: состояние, сохранение поиска, выбор текущего, удаление, очистка истории отправок, переключение статуса и запуск поиска/предпросмотра.
- Корень Vercel `/` и страницы `/ui`, `/index`, `/index.html` теперь открывают веб-панель, а служебные JSON endpoints оставлены на `/status`, `/health`, `/api/*`.
- Веб-панель переработана по структуре: вместо рыхлого набора блоков теперь есть верхняя панель, статусная лента, быстрые действия, отдельный список сценариев и отдельный редактор сценария с группировкой фильтров по смыслу.
- Если данные панели не загрузились, экран больше не зависает на `Загрузка...`: показывается явная ошибка API и подсказка, что обновить или проверить.
- В пользовательских интерфейсах зафиксирована единая терминология: `сценарий`, `активный сценарий`, `проверить вакансии`, `автопоиск`; старые смешанные подписи вроде `текущий поиск` и `поиск сейчас` нужно считать устаревшими.
- При проверке Vercel важно различать production URL и старые preview-ссылки формата `...git-main...vercel.app`: после CLI production deploy они могут показывать старый интерфейс, даже если основной домен уже обновился.
- Стартовое сообщение Telegram-бота при ручной проверке и предпросмотре больше не должно показывать название сценария как будто это поисковый запрос: теперь оно строится из самих параметров поиска (`запрос`, `формат`, `география`, `исключения`).
- Веб-панель дополнительно переработана под структуру официального HH: слева сценарии и фильтры, справа активный сценарий, статусы, результаты и пагинация.
- Блок результатов теперь визуально отделён и всегда находится в правой рабочей зоне; пустое состояние, ошибки загрузки и навигация по страницам должны быть видны сразу без поиска по экрану.
- Для длинных текстов добавлены более жёсткие ограничения по переносам и прокрутке внутри карточек и сводок, чтобы названия, summary и выдача не вылезали за края блоков.
- Поисковый движок HH ускорен: запросы к комбинациям `запрос + опыт` теперь выполняются параллельно с ограничением по воркерам, а не полностью последовательно.
- Для Telegram API и HH API включено повторное использование HTTP-сессий с пулом соединений, чтобы не тратить время на постоянное создание новых сетевых подключений.
- Добавлен короткий кеш результатов одинакового поиска (`HH_SEARCH_RESULT_TTL_SECONDS`, по умолчанию 90 секунд): повторные запуски из сайта и бота должны отрабатывать заметно быстрее.
- Пустые ответы с одними ошибками не кешируются, чтобы временные сетевые сбои не залипали в ускоренном кеше.
- Ручной запуск и автопоиск в Telegram больше не ждут, пока соберётся вся первая страница: первые вакансии отправляются сразу по мере нахождения, в пределах выбранного размера страницы выдачи.
- После живой отправки бот всё равно сохраняет постраничный список результатов; если найдено больше, чем показано сразу, в финальном сообщении появляется кнопка открытия списка.
- При live-выдаче приоритет отдан уже показанным вакансиям: если их успели отправить сразу, они должны остаться и в сохранённой итоговой выдаче, даже при сортировке и лимите результатов.
- Ранжирование по совпадению ускорено: подготовленные поисковые запросы и их токены теперь кешируются в памяти, чтобы не нормализовать и не разбирать одни и те же фразы заново для каждой вакансии.
- Статические справочники панели (`опыт`, `формат работы`, `сортировка`, интервалы и т.д.) теперь кешируются в памяти процесса, чтобы открытие веб-панели не тратило время на их сборку при каждом запросе.
- Пользовательскую терминологию бота и веб-панели снова развели по смыслу: `шаблон` — сохранённый набор фильтров, `текущий шаблон` — выбранный шаблон для работы, `автопроверка` — его автоматический запуск по расписанию.
- Раздел `Мои поиски` заменён на `Мои шаблоны`, а `Активный поиск` заменён на `Текущий шаблон`; старые подписи нужно считать устаревшими и больше не возвращать в интерфейс.
- Мастер создания и редактирования шаблона теперь должен явно объяснять, что вопросы идут по одному блоку, а в конце всегда будет итоговое подтверждение перед сохранением.
- На текстовых шагах редактирования шаблона разрешено оставлять текущее значение через `ok`: это нужно для запросов, регионов, слов и работодателей, чтобы не заставлять пользователя переписывать всё заново.
- Для запросов к HH добавлена защита от временных блокировок: при 403/429 бот теперь делает короткий backoff, повторяет запрос и притормаживает остальные параллельные запросы, вместо мгновенного завала всей выборки.
- Для тяжёлых шаблонов поиск теперь сам снижает число параллельных воркеров в зависимости от нагрузки (`количество комбинаций × страниц`), чтобы уменьшить риск антибот-ограничений HH.
- Пользовательские ошибки HH в Telegram и веб-панели теперь показываются в человекочитаемом виде без длинных raw-URL; 403 трактуется как временное ограничение HH, а не как непонятный сбой фильтров.
- В шаблоны добавлены правила `формат по регионам`: теперь можно задать общий формат работы и поверх него отдельные исключения по странам или городам, например `Россия = удалённо`, `Беларусь = удалённо`.
- Правила формата по регионам работают и в backend-фильтрации HH, и в веб-панели; для совпавшего региона локальное правило заменяет общий список форматов работы.
- Дефолтный активный шаблон мигрирован: для Беларуси и России добавлено правило `только удалённо`, а в опыте по умолчанию теперь хранятся `Не имеет значения`, `Без опыта` и `1–3 года`.
- Если вакансии по фильтрам найдены, но скрыты только потому, что уже отправлялись раньше, Telegram-бот теперь показывает отдельную кнопку `Пройтись заново`: она очищает историю текущего шаблона и сразу повторно запускает поиск одним нажатием.
- В дефолтный шаблон аналитики добавлено обязательное включающее слово `python`, и оно проверяется именно в содержимом вакансии (`Только в описании`), чтобы отсеивать нерелевантные вакансии без Python.
- Исправлена логика опыта: если вместе с `Не имеет значения` выбраны конкретные диапазоны (`Без опыта`, `1–3 года` и т.д.), поиск больше не расширяется до любого опыта; приоритет остаётся у конкретных диапазонов.
- Для текущего дефолтного шаблона правило `только удалённо` расширено на `Кыргызстан` и `Узбекистан`; существующие региональные правила при этом не затираются.
- Живая выдача Telegram теперь продолжает список с первой ещё не показанной страницы: после первых 5 вакансий кнопка ведёт дальше по новым вакансиям, а не дублирует уже показанные сразу карточки.
- В навигации списка результатов на последней странице добавлена кнопка `С начала`, чтобы заново пройти весь сохранённый список по кругу; промежуточные страницы по-прежнему дают `Назад` и `Далее`.
- Текущий дефолтный шаблон аналитики переведён на сортировку `По дате размещения`, чтобы сверху всегда шли самые свежие вакансии.
- Финальное сообщение после живой выдачи теперь строит кнопку `Дальше` напрямую из числа уже показанных и оставшихся вакансий, без зависимости от ленивого чтения result-session: если после первых 5 есть продолжение, кнопка обязана появиться.
- При повторном нажатии `Проверить сейчас` бот больше не должен заново показывать первые вакансии из незавершённой выдачи: если остались непросмотренные страницы прошлого списка, он продолжает именно их.
- Состояние бота теперь сохраняется не только в Vercel Runtime Cache, но и в локальный `/tmp`-файл процесса; это уменьшает риск потери истории отправок и повторного показа уже выданных вакансий между запросами.
- После достижения лимита живой выдачи бот теперь сразу отправляет промежуточное сообщение `Показал первые N вакансий. Собираю продолжение списка...`, а затем редактирует именно его в итог с кнопкой `Дальше`; это убирает ситуацию, когда управляющее сообщение теряется ниже по чату.
- В пагинации результатов Telegram убрана вторая дублирующая кнопка `Назад`: теперь верхний ряд отвечает только за перелистывание страниц, а нижний — за выход в текущий шаблон или главное меню.
- В промежуточное сообщение после первых найденных вакансий добавлены кнопки `Текущий шаблон` и `Главное меню`, чтобы навигация не пропадала, даже если итоговая клавиатура с `Дальше` приходит позже.
- Для Vercel Python-функций увеличен `maxDuration` до `300`, чтобы тяжёлый поиск чаще успевал завершиться и отредактировать промежуточное сообщение в финальный итог с кнопкой `Дальше`.
- Уже показанные в живой выдаче вакансии теперь записываются в историю сразу в момент отправки, а не только после завершения полного поиска; это убирает повтор первых вакансий при повторном нажатии `Проверить сейчас`, если предыдущий длинный прогон ещё не успел полностью закончиться.
- Живая Telegram-выдача больше не ждёт окончания всего поиска, чтобы открыть продолжение: как только у бота уже накопилась вторая страница результатов, он сохраняет промежуточную result-session и добавляет кнопку `Дальше` прямо в текущее сообщение `Показал первые N вакансий...`.
- Telegram-команды `Проверить сейчас` и `Предпросмотр` на Vercel переведены на неблокирующий запуск через отдельный self-call `/run-now`: webhook-ответ больше не должен удерживаться на всё время длинного поиска, поэтому кнопка `Дальше` может открываться, пока основной поиск ещё продолжается.
- Запуск async-поиска на Vercel переведён с daemon-thread на прямой self-call с коротким `read timeout`: это надёжнее для serverless-среды и не должно больше приводить к ситуации, когда после сообщения `Проверяю вакансии` поиск вообще не продолжался.
